
Backend Physical Design using OpenLane

Overview

This branch contains the backend physical design flow for the Custom Audio SoC project.
The backend implementation uses the open-source RTL-to-GDSII flow provided by OpenLane, which integrates multiple ASIC design tools from the OpenROAD Project.

The objective of this backend flow is to transform the verified RTL design of the SoC into a physically realizable chip layout, including floorplanning, placement, clock tree synthesis, routing, and final GDS generation.

⸻

Design Flow

The backend design flow implemented in this branch follows the standard ASIC physical design pipeline:
	1.	Synthesis
	•	Converts RTL (Verilog) into a gate-level netlist using standard cell libraries.
	2.	Floorplanning
	•	Defines chip core area, aspect ratio, and IO placement.
	•	Establishes power distribution and macro placement.
	3.	Placement
	•	Places standard cells optimally within the core area.
	•	Optimizes timing, congestion, and power.
	4.	Clock Tree Synthesis (CTS)
	•	Builds the clock distribution network.
	•	Minimizes clock skew and insertion delay.
	5.	Routing
	•	Connects all placed cells through metal interconnect layers.
	6.	Sign-off Checks
	•	Design Rule Check (DRC)
	•	Layout Versus Schematic (LVS)
	•	Static Timing Analysis (STA)
	7.	GDSII Generation
	•	Produces the final layout file used for chip fabrication.

Requirements

To run the backend flow, install:
	•	Docker
	•	Python 3.x
	•	Git

Install the OpenLane environment:

git clone https://github.com/The-OpenROAD-Project/OpenLane.git
cd OpenLane
make setup


⸻

Running the Backend Flow
	1.	Navigate to the OpenLane directory:

cd OpenLane

	2.	Run the OpenLane flow:

make mount

	3.	Start the OpenLane interactive shell:

./flow.tcl -interactive

	4.	Load the design configuration:

prep -design <design_name>
run_flow


⸻

Expected Outputs

After successful execution, OpenLane produces:
	•	Gate-level netlist
	•	Placement and routing reports
	•	Timing analysis reports
	•	Final GDSII layout

These outputs are located inside the runs/ directory generated by OpenLane.

⸻

Design Target

The backend flow is designed for the Custom Audio SoC, which includes:
	•	32-bit RISC-V processor
	•	Memory-mapped PWM audio peripheral
	•	Interrupt-driven control logic

The goal is to achieve a clean physical implementation ready for fabrication or further ASIC validation.
